# Documentation Index

This folder contains the main project documents for product, technical, and implementation tracking.

---

## 1. Product Documents

### `PRD_v1.0.md`
Product requirements document.

Use this to understand:
- what the product is trying to solve,
- who it is for,
- what is in scope for the MVP.

### `POSITIONING_AND_DIFFERENTIATION.md`
Product positioning and differentiation.

Use this to understand:
- how this product differs from template-based tools,
- why it should evolve as a dedicated desktop application,
- what its long-term product direction should be.

## 2. Technical Documents

### `TECH_DESIGN_v1.0.md`
Technical design document.

### `IMPLEMENTATION_PLAN_v1.0.md`
Phase-based implementation plan.

## 5. Practical Usage

- Read **PRD** when thinking about product scope.
- Read **Positioning** when deciding what kind of product this should become.
- Read **Tech Design** when changing architecture or modules.
- Read **Implementation Plan** when planning work.
- Read **Progress Snapshot** when checking current state.
